# Review Rubric / 评审标准

## Verdict values

- `likely_correct`：在当前假设下大概率正确，主要剩证明/细节。
- `salvageable`：方向中有可用部分，但需要 pivot 或补条件。
- `high_risk`：不一定错，但正确性/复杂度/实现风险明显。
- `likely_wrong`：存在疑似致命问题或强反例方向。
- `insufficient_info`：信息不足，无法给出可靠判断。

中文输出时可对应：

- 大概率正确
- 方向可救
- 高风险
- 大概率错误
- 信息不足

## Risk scores

风险分数为 1 到 5。

### Correctness risk / 正确性风险

1. 证明路径清楚。
2. 有细节缺口，但大概率是标准补全。
3. 有一个关键 lemma 需要认真检查。
4. 多个核心断言未证明。
5. 很可能被小数据或极端情况打掉。

### Complexity risk / 复杂度风险

1. 复杂度明确且安全。
2. 可能有轻微隐藏常数/对数。
3. 依赖非平凡摊还或边界估计。
4. 声称复杂度可疑。
5. 明显超过约束。

### Implementation risk / 实现风险

1. 简单模板级。
2. 有少量边界。
3. 中等复杂度，需要认真写。
4. 长且脆，容易出现状态/边界 bug。
5. 比赛中不推荐，除非没有替代方案。

## Contest writability / 比赛可写性

- `recommended` / 推荐：适合作为比赛实现方案。
- `acceptable` / 可接受：可以写，但要小心。
- `risky` / 有风险：可能耗时较长或 bug 较多。
- `not_recommended` / 不推荐：除非必要，否则不应优先写。

## 评审优先级

1. 这个想法是否真的回答了题目？
2. 是否用到了主要约束和识别信号？
3. 正确性证明是否可信？
4. 复杂度是否安全？
5. 是否存在更简单、更比赛可写的方案？
6. 下一步最该检查什么？

## 空项处理

模板空项不等于错误。若信息缺失：

- 继续评审已有内容；
- 降低置信度；
- 在“缺失信息影响”中说明最多 1~3 个最关键缺失项；
- 不要要求用户补全所有栏目。
