# Misjudgment Tags / 误判标签

用于标记一次口胡为什么薄弱或错误。每题建议 1~3 个标签，不要滥用。

## 核心标签

### `type_misclassification` / 题型误判

第一反应把题放进了错误家族。

例：

- 把贪心/构造题想成 DP；
- 把贡献统计题想成复杂数据结构维护；
- 把缩点/连通性题想成最短路。

### `overengineering` / 过度设计

算法可能对或部分对，但明显比必要做法更重。

常见表现：

- 状态空间很大但必要性不清；
- 能排序/离线/拆贡献，却上了重数据结构；
- 能用简单不变量，却上了高级数学。

### `missed_property` / 漏性质

题面或约束里的关键性质没有被使用。

常见性质：

- 单调性；
- 可交换性；
- 独立性；
- 局部到整体；
- 树/区间嵌套；
- 可逆性；
- 贡献可分离。

### `proof_gap` / 证明缺口

思路依赖未经证明的断言。

需要警觉的句式：

- “显然最优”；
- “贪心选这个”；
- “顺序不影响”；
- “这些事件独立”。

### `complexity_misread` / 复杂度误读

复杂度估计不完整或不真实。

例：

- 隐藏多一维/一层 log；
- 摊还没有证明；
- 每个状态转移不是 O(1)；
- 预处理过大。

### `implementation_risk_misread` / 实现风险低估

理论上可行，但比赛里太长、太脆或太难 debug。

常见表现：

- 多个状态相互影响；
- rollback/persistence/lazy 很复杂；
- 精度/取模/边界很脆；
- 不变量很难检查。

### `template_bias` / 模板偏置

熟悉模板把思考带偏。

例：

- 强行 NTT / 生成函数；
- 强行 SAM / 后缀数组；
- 强行线段树 beats；
- 强行费用流。

### `counterexample_blindness` / 反例意识不足

没有用小数据、极端数据、退化情况主动攻击自己的想法。

### `missing_constraints` / 忽略约束

没有充分利用或检查数据范围、时限、内存、输入生成方式等。

### `wrong_priority` / 优先级误判

一个方向可能可做，但不是比赛中最该优先写/想的方案。
