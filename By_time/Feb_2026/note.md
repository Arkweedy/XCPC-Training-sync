## 1625D
首先得记起来一个集合中的两两xor最小值可能是哪些pair的xor:
排序后相邻的xor
这可以认为是01trie上的lca处取到，其实比较好理解
所以这题可以边维护一个trie，边贪心/dp
trie维护前缀为从根到节点的路径时， 满足条件的集合的最大大小和对应的数(这个方便我们倒着推回去构造方案)
我们只需要做到两件事：
1. 查询加入目前这个数， 合法的最大集合和这个集合的最大数(因为我们是从小到大插入，实际上是维护最后加入这个集合的数)
2. 插入这个以这个数结尾的集合到trie

查询操作使用贪心的思想，从大到小遍历$k$的位,同时在trie树上走维护与当前数的xor：
1. 如果是 $0$， 那么我走xor到变成$1$的节点，此时xor一定大于$k$， 这个节点代表的集合均可以，我们利用我们维护的最大值和id更新答案
2. 如果是 $1$， 我们必须走到xor变成$1$的节点， 且子节点不一定能都成立，我们不能更新答案
3. **走到最后一位，此时xor的值等于k**, 按照题意这也是合法的，用这个点的信息更新答案。

插入很简单，一路更新ma和id即可

最后根据每次查询得到的pre维护一个pre表，以及当前最大结合的最大节点， 倒着找回去即可。

## 2009G3
很好的一道题，做法好像也比较多，有用历史和什么的，但是这种数据结构科技还是先不接触了
也有大分讨+矩形区间和去做的，感觉用到的东西比较重型
这里记录一下基于维护suf/pre链（高观点一些应该是笛卡尔树？）的做法， 而且这个做法的时间复杂度取决于rmq， 所以甚至可以做到$O(n + q)$

先分析一下要求的函数$f(l, r)$, 由于固定了区间长度为$k$， 那么答案一定是区间内若干个长度为 $k$的窗口的答案的最小值。
我们考虑做一个滑动窗口的预处理，将$a[n]$ 变成 $b[n - k + 1]$,其中 $b_i = f(i, i + k - 1)$，询问的 $r$ 也变成 $r - k + 1$ , 那么有：
$$f(l,r) = \min_{i = l}^{r} b_i$$
我们要求的是 $F(l, r) = \sum_{i = l}^{r} \sum_{j = i}^{r} f(l, r)$
对于这种区间最小值的贡献，考虑一个经典的拆贡献方法：我们找到区间最小值的位置$x$,那么它的贡献是$$b_x(x - l + 1)(r - x + 1)$$
而剩下的贡献被分成两个独立的区间。
当然，我们并不能递归求解，这样复杂度不对。
不过这个方法仍然有扩展性。

在此之前，我们先明确定义区间最小值为最左的最小值，这样就可以唯一确定。
随后定义一个位置的 $suf$ 和 $pre$, $suf_i$ 是 $b_i$ 后第一个严格小于 $b_i$的位置, 而 $pre_i$ 则是 $b_i$ 前第一个小于等于 $b_i$ 的位置。
这样定义是为了精确划分贡献， 使得当我们使用$pre_i$和$suf_i$来表示一个数的贡献区间时，刚好可以不重不漏地覆盖所有的区间。

再回到这个方法， 我们用区间最小值 $b_x$ 分割了区间 $[l, r]$， 那么怎么求解子区间 $[l, x - 1], [x + 1, r]$呢？
考虑求解左区间$[l, x - 1]$
我们定义
$$g(i, t) = \sum_{j = i}^t f(i, j)$$
那么$F(l, x - 1) = \sum_{i = l}^{x - 1} g(i, x - 1)$
考虑$b_i$的贡献，可以发现，$g$ 可以通过 $suf$ 递推：
$$g(i, t) = (suf_i - i)b_i + g(suf_i, t)$$
于是我们可以$O(n)$递推求解$g(*, t)$
但是这样还不够，我们希望我们可以$O(1)$求解这个和式， 而这个和式是一段连续的和， 很容易想到前缀和，但是$t$是随$x$变化的，怎么办呢？
考虑$g(i, j) = g(i, k) - g(j + 1, k), k > j$， 当 $j + 1$ 是 $i$ 的若干次 $suf$
证明考虑将 $g(i, k)$ 展开：
$$g(i, k) = \sum_{t} (suf^t_i - suf^{t - 1}_i)b_{suf^{t - 1}_i} + g(j + 1, k)\\ = g(i, j) + g(j + 1, k)$$
于是我们可以预处理参数$k = n + 1$(1-base默认$suf_n = n + 1, pre_n = 0$)的 $g_{n + 1}, h_0$ 数组
而如果我们选择用区间最小值$a_x$来分割区间为前后缀两个区间， 那么这个区间最小值一定是$a_l \dots a_{x - 1}$的若干次$suf$ 和 $a_{x + 1}, a_r$的若干次 $pre$
计算$g(i, j)$不是问题后，我们再推一下$F$的式子：
$$\begin{align*}
F(l, x - 1) &= \sum_{i = l}^{x - 1} g(i, x - 1)\\
&= \sum_{i = l}^{x - 1} g_{n + 1}(x) - g_{n + 1}(i) \\
&= (x - l)g_{n + 1}(x) - \left(\sum_{i = l}^{x - 1}g_{n + 1}(i)\right)
\end{align*}$$
显然后面那个求和，我们对$g_{n + 1}$做一次前缀和就可以$O(1)$差分出来。
那么我们就可以使用我们$O(n)$预处理出来的$g,h$等数组$O(1)$地计算出$F(l, x - 1), F(x + 1, r)$
于是我们可以$(n\log n)$预处理(主要是最小位置的rmq查询，其实可以做到$O(n)$),$O(1)$回答单次询问
总时间复杂度$O(n\log n + q)$



## 2203F
一道很有意思的题，可以加强对二叉搜索树结构的理解
在这题定义下的一些事实:
1. 二叉搜索树只有$\log_2 n$ 层， 
2. 二叉搜索树每层子树大小最多相差 $1$
3. 二叉搜索树的子树大小只有 $\log n$ 种
4. 若子树大小为 $n$, 左子树大小为$\lfloor\frac{n - 1}{2}\rfloor$, 右子树大小为$\lceil\frac{n - 1}{2}\rceil$

我们回到这一题，要计算`swap(i,j)`,搜得到$x$个点的情况各自有多少种,先推一下式子, 不妨设$i < j$：
1. 如果$\operatorname{lca}(i, j) = i \text{ or } j$, 那么搜不到的有$|i - j|$个,他们是$i,j$之间的所有数($|i - j| - 1$ 个)， 和$\operatorname{lca}(i, j)$。
2. 如果$\operatorname{lca}(i, j) = k, k \neq i,j$, 那么搜不到的有:$i, j$， 以及$i$的右子树内的点， $j$的左子树内的点

那么我们可以把贡献分成两类去计算
有2种方法都可以通过该题，1种复杂度是$O(nlogn)$， 比较极限。
第二种是$O(n + \log^3 n)$, $n = 6e5$仅需300ms

### 第一种
我们考虑根据BST序来枚举$i, j$
预处理每个点作为根的$sz$和$fa$
做法是枚举当前点为$j$ ，同时用map存根据$sz$分类的，先前遍历过的点的信息，而$sz$只有$\log$种， 整个遍历下来是$O(n\log n)$。
然而，这会将情况1也当作情况2算， 于是我们再修正一下就可以了，我们对于每个点都暴力搜完整个子树(或者跳到根)来枚举情况2进行修正， 这部分的复杂度为$O(n\log n)$。

### 第二种
我们将两种情况分开算，先枚举$\operatorname{lca} = k$， 再枚举$i, j$
$k$ 作为子树根， $i, j$只能是左右子树内的点(或者 $k$ 本身)
注意本质不同的子树形态只有 $\log$ 种， 我们完全可以对这 $\log$ 种子树分开计算贡献，。
首先我们需要预处理每种 $sz$ 的子树有多少个， 这个可以用map维护一个队列进行预处理

对于情况1，我们需要选择子树根， 和左/右子树中的一个点。
需要注意的是，左右子树的点的点值是连续的一个区间，左子树是$[k - sz(ls(k)), k - 1]$, 右子树是$[k + 1, k + sz(rs(k))]$, 那么贡献区间也很清楚了，就是$[1, sz(ls(k))]$ 和 $[1, sz(rs(k))]$。
这里没必要差分，暴力加的总复杂度是$O(n)$的。

对于情况2，我们需要知道大小为$sz(k)$的子树中，各有多少个不同$sz$的子树， 这个我们只需要自底向上地维护不同$sz$的子树的map即可。
求解时，二重循环枚举左右子树内的子树大小$a, b$ ，贡献位置为$rsz(a) + lsz(b) + 2$

核心代码如下：
```c++
for(auto [k, c] : v){ 
    int l = (k - 1) / 2, r = k / 2;
    for(int i = 1;i <= l;i++){
        ans[i] += c;
    }
    for(int i = 1;i <= r;i++){
        ans[i] += c;
    }
    for(auto [x, c1] : mp[l]){ 
        for(auto [y, c2] : mp[r]){
            ans[rsz(x) + lsz(y) + 2] += 1ll * c1 * c2 * c; 
        } 
    }
    mergeto(l, r, k);
    mp[k][k]++;
}
```


## 1736E
一道小清新dp，不过我的dp功底好像还是偏差了些
设置状态 $dp_{i,j,k}$ 为前 $i$ 个数， 第 $i$ 个数是 $a_j$ , swap了 $k$ 次的最大贡献, 根据题目限制， $k \leq i$
考虑一个数只会先往左，再往右
对第 $i$ 个数是从左边/右边转移过来讨论：
从 $i - 1$ 转移过来：
$$dp_{i, j, k} = dp_{i - 1, j, k - 1} + a_j$$
从 $j \geq i$ 转移过来：
$$dp_{i, j, k} = \operatorname{max}\limits_{t < j} \{dp_{i - 1, t, k - 1}\} + a_j$$

    对于第一类转移， 直接转移即可。
    对于第二类转移， 使用前缀优化即可。
时间复杂度$O(n^3)$
