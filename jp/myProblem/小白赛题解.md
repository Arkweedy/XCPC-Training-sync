# C 至圣的舞萌
哈希表去重取最高分、固定容量小根堆维护前50大
思路概括
1. 使用哈希表维护每首曲目的最高分。
2. 在线把不同曲目的最高分插入容量为 50 的小根堆，堆满后仅当新值大于堆顶时替换，以维护前 50 大。
3. 最后把堆中元素求和；当种类不足 50 时，堆中即为全部最高分。
时间复杂度分析

哈希插入查找为摊还 O(1)。对 m 个不同曲目做堆操作，单次 O(log 50)，总体 O(m)。总复杂度 O(n + m)。

# E 夜羽的神秘仪式
可以对每个数使用 $O(\sqrt n)$ 的时间筛出因子,每个因子使用 $O(\log k)$的时间计算。
总的时间复杂度为 $O(n \sqrt n + n \log n \log k)$ , $10 ^ 6$的数据应该可以不太能通过。
考虑使用以下的部分优化应该可以通过：
使用枚举倍数的方法枚举因子可以将时间复杂度降到 $O(n \log n \log k)$
使用快速幂预处理出来自然数的 $k$ 次幂，可以将时间复杂度再降为 $O(n \log n + n \log k)$
枚举因子的时候，直接计算每个因子出现了多少次，时间复杂度可以到 $O(n \log k)$

其实这题本来还有一个Hard_version,回答 $n = 5 \times 10 ^ 6$ 下的 $q = 5000$ 次询问
下面是Hard_version 的题解： 

首先要对预处理自然数的 $k$ 次幂做优化。
自然数 $k$ 次幂函数 $\mathrm{id}^k$ 是一个完全积性函数，可以使用欧拉筛 $O(\frac{n \log k}{\ln n}) \sim O(n)$预处理出来。
预处理的思路是在质数处使用快速幂计算，单次计算$O(\log k)$.
根据质数密度公式，$[1 , n]$ 中有 $O(\frac{n}{\ln n})$ 个质数，计算的复杂度约为 $O(n)$
对于合数，在欧拉筛枚举出它的时候，合数被分解成它的最小质因子和另外一部分的乘积，这些都处理过了，于是可以 $O(1)$ 计算出来。这部分的总复杂度是 $O(n)$ 的。

对于多次询问怎么回答，一个好的想法是计算出每个位置的 $f(i)$,求个前缀和后差分 $O(1)$ 回答。
然而在给定的数据范围，如果使用我们在Easy Version的求法，复杂度是 $O(n \log n)$,不太好通过。

一个思路是继续延续前面求出每个位置的 $f(i)$， 求前缀和的想法，但是使用更快的求 $f(i)$ 的方法。
使用**狄利克雷前缀和**，这是一种将每个质数看成一维，求高维前缀和的思维方式。通过狄利克雷前缀和加速，可以将累计因子的贡献(即这题的求解$f(i)$) 加速到 $O(n \log \log n)$,剩下的 $O(1)$ 回答$q$ 次询问即可。

另外一个思路是考虑我们前面的一个优化—— 计算因子的贡献时，直接计算因子出现了多少次。
实际上，每个数 $x$ 作为因子的贡献次数是 $\lfloor\frac{n}{x}\rfloor$ 次。
进一步地，我们观察到这是可以整除分块的，于是可以预处理 $\mathrm{id}^k$ 的前缀和后，使用整除分块$O(\sqrt{n})$ 回答单次询问。

最后的时间复杂度：
欧拉筛预处理 $\mathrm{id}^k$： $O(n)$
狄利克雷前缀和做法 : $O(n \log \log n + n + q)$
整除分块做法 ： $O(n + q \sqrt{n})$



# F 至圣打东方
1. 将每个格子视作图的一个点：相邻四方向在原始为空地的格子之间连一条权重为0的边；

2. 若在某格子释放一次《梦想封印》，则以该格子为中心、曼哈顿距离≤k 的所有格子都会被清除并可通行；这相当于从该中心向这些格子连一条权重为1的边；

3. 用0-1 BFS在网格上求从起点到终点的最小“释放次数”。为避免重复从同一中心多次释放，设 released_done[x][y] 标记每个格子的“释放扩展”仅处理一次；

4. 起点若为‘#’，仍可在起点先释放一次（代价1）再开始移动；若为‘.’，则可以先进行0代价的连通块扩展，再按需释放。

  

时间复杂度分析

单次“释放”扩展枚举一个菱形，大小约为O(k^2)。每个格子至多触发一次释放，故最坏复杂度约为O(n*m*k^2 + N)。在n,m<100,k≤20的约束下可通过。
