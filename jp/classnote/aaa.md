一、分析题
功能差异:
take xs i：返回前 i 个元素，且保持原顺序。
rtake (xs, i, taken)：把前 i 个元素压入 taken 的表头，因此返回的是前 i 个元素的逆序（若初始 taken=[]）。
想得到与 take 相同的结果，可用 rev (rtake (xs, i, []))，多一次 rev 的 O(k)（k = min(|xs|, i)）开销。
性能
设 k = min(|xs|, i)。
take：每步做 O(1)，共 k 步 → work = Θ(k)。但非尾递归（x :: take ... 之后还有 cons），栈深 = Θ(k)，大输入可能栈溢出。
rtake：每步做 O(1)，共 k 步 → work = Θ(k)。是尾递归，栈深 = O(1)，实现上更稳健。
若要 rev (rtake...) 来保持顺序，则总 work = Θ(k) + Θ(k) = Θ(k)（常数略大），栈仍 O(1)。
结论：
只要能接受“逆序”，rtake 更好（尾递归、常量栈）。
若需要保持顺序：rev (rtake ...) 与 take 都是 Θ(k) work，但前者栈 O(1)，更安全。

二、求值过程：nextperm [2,3,1,4]
逐步计算（每步用 =>）：

nextperm [2,3,1,4] => next ([2], [3,1,4])

hd [2] = 2，与 y = 3 比较，2 < 3 成立
=> next (3::[2], [1,4]) => next ([3,2], [1,4])

hd [3,2] = 3，与 y = 1 比较，3 < 1 不成立，走 swap：

xlist = [3,2]，ys = [4]

swap (3::2::[])：

看 2 > y(=1)，成立
swap (3::2::[]) => 3 :: swap (2::[])

swap [2] => y::2::ys = 1::2::[4] = [1,2,4]

于是 3 :: [1,2,4] = [3,1,2,4]

所以第 3 步：
=> [3,1,2,4]

最终结果：nextperm [2,3,1,4] = [3,1,2,4]。
