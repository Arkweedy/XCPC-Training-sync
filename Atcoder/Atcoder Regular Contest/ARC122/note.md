# E
先考虑怎么形式化约束：
约定符号$\operatorname{lcm}_i = \operatorname{lcm}_{j \leq i}\{a_j\}$
要求$$\operatorname{lcm}_i < \operatorname{lcm}_{i + 1} = \operatorname{lcm}(a_{i + 1},\operatorname{lcm}_i)  = \frac {a_{i + 1}\operatorname{lcm}_i}{\gcd(a_{i + 1},\operatorname{lcm}_i)}\\ \Rightarrow a_{i + 1} > \gcd(a_{i + 1}, \operatorname{lcm}_i) = \operatorname{lcm}_{j < i} \{\gcd(a_i,a_j)\}$$
或者也可以写$a_{i + 1} \nmid \operatorname{lcm}_i$
实际上右式最大只能等于左式
于是我们得到了一个check一个数能不能放到当前构造的末尾的方法，不会受很大的数不容易求解$\operatorname{lcm}$的影响。
显然我们不能正着构造，这样需要回溯，时间复杂度是阶乘级的。
如果倒着构造，我们还需要在失败时回溯吗？
考虑倒着构造，我们每一步都有一个合法集 $S$，那么我们的每一步构造就是尝试从 $S$ 中选取一个数放到构造序列的开头，并且得到一个新的 $S'$ ,而当 $S$ 为空时则证明构造结束/失败。
那么选取数的顺序会造成影响吗？
考虑我们选择了一个数$x$， 显然约束减少了$\gcd(a_i,x)$这一项，那么$S$中原本有的数不会被移除。
有一些数会因为约束的减少新加进来，虽然这一次新加进来的数确实和选择的$x$有关，但是我们考虑如果我们移除$S$中的一个子集$T$后一个数$y$可以加入$S$, 那么$y$一定会加入$S$, 因为$S$最终一定会变成空集(无论是否有解)，我们一定是移除了$T$中的所有数。这种感觉像是某种拓扑排序之类的东西。如果我们考虑$A$是我们构造的序列中出现的元素集合，那么$A$一定会扩充为唯一的最大集(可以类比传递闭包？)， 显然有解当且仅当$|A| = n$。
于是我们可以倒着枚举，如果发现可以作为末尾的数就立刻加入构造序列，复杂度$O(n^3\log V)$， 实际上常数应该相当小，因为我们遇到第一个合法的就会终止循环。

实际上倒着构造这一思路也可以由归纳法得出
